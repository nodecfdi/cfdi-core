createDocument

  • createDocument(namespaceURI, qualifiedName, doctype?): Document

  • Creates an XML Document object of the specified type with its document element.

    It behaves slightly different from the description in the living standard:

    • There is no interface/class XMLDocument, it returns a Document instance (with it's type set to 'xml').
    • encoding, mode, origin, url fields are currently not declared.

    Parameters

    • namespaceURI: null | string
    • qualifiedName: string
    • Optionaldoctype: null | DocumentType

    Returns Document

    The XML document.

createHTMLDocument

  • createHTMLDocument(title?): Document

  • Returns an HTML document, that might already have a basic DOM structure.

    It behaves slightly different from the description in the living standard:

    • If the first argument is false no initial nodes are added (steps 3-7 in the specs are omitted)
    • several properties and methods are missing - Nothing related to events is implemented.

    Parameters

    • Optionaltitle: string | false

    Returns Document

    See

    • DOMImplementation.createDocument

hasFeature

  • hasFeature(feature, version?): true

  • The DOMImplementation.hasFeature() method returns a Boolean flag indicating if a given feature is supported. The different implementations fairly diverged in what kind of features were reported. The latest version of the spec settled to force this method to always return true, where the functionality was accurate and in use.

    Parameters

    • feature: string
    • Optionalversion: string

    Returns true

    Deprecated

    It is deprecated and modern browsers return true in all cases.
